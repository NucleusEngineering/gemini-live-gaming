System Architecture

C# Scripts for screen capture, audio streaming, and receiver.

Web-based interface for screen share and live chat history.

Backend: The Bridge

  • Built with Python FastAPI and Uvicorn.
  • Asynchronous WebSocket handling for low-latency streaming.
  • Orchestrates Gemini sessions with resumption tokens for stability.
  • Manages queues for audio, video, and text inputs separately.
async with client.aio.live.connect(model=MODEL) as session: ...

RAG: Real-time Project Context

  • Dynamic file indexing using File Search Stores in Gemini.
  • Tools enabled: get_project_context and read_project_file.
  • Gemini can "browse" your scripts to understand bug reports in full context.
  • Automatic syncing of documentation files (e.g., STORYLINE.md).

Unity: The Primary Client

  • ScreenStreamer: Efficient camera render to JPEG (1 FPS).
  • AudioStreamer: Captures system microphone to PCM 16kHz.
  • AudioReceiver: Real-time playback of Gemini's vocal response.
  • GeminiLiveClient: Core WebSocket manager for Unity.

Tool Calling: Natural Language → Action

1. Decision

Gemini hears "Jump" and calls control_movement(action="JUMP").

2. Bridge

Backend forwards the tool event to Unity via WebSocket.

3. Execution

Unity's GeminiLiveClient triggers a C# Action.

OnToolCallReceived?.Invoke(message.name, message.args?.action);

Enables complex interactions like controlling characters or modifying scene state via voice.

Efficiency: Frame Skipping

How we minimize bandwidth while maintaining multimodal understanding:

if (base64 !== this.lastFrameData) {
  this.ws.send(JSON.stringify({ type: "video", data: base64 }));
}
  • The system hashes the current screenshot and compares it to the last.
  • If nothing changed (static editor), no video data is sent.
  • Saves significant bandwidth and model processing tokens.

Video
